# Repository Guidelines  

This repository maintains the official definitions of the Avian Knowledge Network (AKN) data sharing levels.

## Project Structure & Module Organization
The current definition lives in `main.md`, while `README.md` gives the project overview. Use `historic/` for dated 
specifications, preserving filenames such as `historic/2018-2021_AKN Data Sharing Levels - Google Docs.pdf` to keep 
provenance clear. Place any new reference assets alongside their year-stamped peers inside `historic/`, and keep the 
repository root focused on the canonical definition and contributor docs.  

`how-to-change.md` -  this document has the policy for changing `main.md` by committee.
